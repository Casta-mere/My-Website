### 开发流

笔者的开发流如下:

Dev 版部署于家里的 nas，在公司用 Tailscale 连接；家里直连。每次更新后，本地先 `npm run build` 测试，没问题后直接把 build 好的内容 scp 到服务器上，服务器端直接运行 `npm run serve`。Github 仅用于版本管理，还未尝试 Github Pages 部署

这样的好处如下：

- 不论在哪里，Dev 版都是同一版，上下班无缝衔接
- 服务器端不需要 build，对性能要求低(目前服务器已经 build 不动了)

### SSL 证书

使用阿里云的**个人测试证书**，虽然免费，但是最近开始每次只能续三个月，更新还是比较烦的

## 📊 数据统计

本站的数据统计使用 [Umami](https://umami.is) 搭建，数据展示于 [Umami Dashboard](https://umami.castamerego.com/share/EplxZVQRe6OkyBt3/castamerego.com)

统计包括以下数据:

- 访问量 (Page Views)
- 访客数 (Visitors)
- 跳出率 (Bounce Rate)
- 访问时长 (Duration)
- 特定链接访问量
- 等等

一个好处是可以在站内通过 API 获取数据并展示，比如: 小站已运行 <DaysCounter startDate="2023-09-21"/> 天，累计获得了 <UmamiTotalViewCounter /> 次访问

借此，笔者还在每个 blog 和 doc 页面显示了阅读量数据
